# CityLink — сучасний сайт інтернет-провайдера

Повнофункціональний аналог сайту cl.in.ua, перебудований на сучасному стеку
(Vite + React 18 + Tailwind CSS) з темним дизайном і робочою бізнес-логікою.

## Запуск проєкту

```bash
npm install
npm run dev      # режим розробки → http://localhost:5173
npm run build    # збірка у dist/
npm run preview  # перегляд зібраної версії
```

Збірка створює **один файл `dist/index.html`** (усе вбудовано) — його можна
відкрити подвійним кліком у браузері без сервера або залити на будь-який хостинг.

## Сторінки

- **Головна** — герой, переваги, тарифи, кроки підключення, новини
- **Тарифи** — картки тарифів + інтерактивний калькулятор вартості
- **Новини** — список із пошуком і фільтром за категоріями + детальна сторінка
- **Оплата** — 4 способи оплати з покроковими інструкціями
- **Допомога** — налаштування роутерів (Netis, TOTOLINK) + FAQ
- **Контакти** — контактні дані + форма зворотного зв'язку
- **Заявка на підключення** — форма з валідацією та підтвердженням
- **Особистий кабінет** — демо-вхід, баланс, тариф, історія, поповнення

## Бекенд (Supabase)

Адмінпанель, кабінет абонента, вхід за IP та динамічні тарифи працюють через
**Supabase** (Postgres + Edge Functions + Auth). Деталі — у `supabase/`:

- `supabase/migrations/0001_init.sql` — схема, RLS, RPC, сід-дані.
- `supabase/functions/` — `subscriber-login`, `subscriber-ip-login`, `subscriber-me`.

Налаштування середовища — `.env` (див. `.env.example`):
`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`. Без них сайт працює у режимі
статичного фолбеку (тарифи з `site.js`, кабінет/адмінка показують відповідне повідомлення).

Деплой бекенда: `npx supabase link --project-ref <ref>` → `npx supabase db push`
→ `npx supabase functions deploy`. Першого адміна призначають у SQL Editor:
`select public.promote_to_admin('admin@cl.in.ua');` (після створення цього
користувача в Supabase Auth).

## Бізнес-логіка

- **Адмінпанель** (`#/admin`): CRUD тарифів (зміни одразу на публічному сайті),
  CRUD абонентів, нарахування/списання коштів, призначення IP-адрес. Доступ —
  лише адмінам (Supabase Auth + таблиця `admins`, захист через RLS).
- **Вхід за IP**: якщо публічна IP абонента збігається з призначеною (`ip_address`),
  кабінет відкривається автоматично без пароля (Edge Function `subscriber-ip-login`).
- **Оплата**: сторінка `Payment.jsx` — сума + номер договору, далі редірект на
  `next.privat24.ua`. Карткові дані на сайті не збираються.
- **Калькулятор тарифу**: підрахунок вартості з урахуванням реальної IP та періоду.
- **Демо-абонент**: логін `1001` / пароль `demo` (створюється сідом міграції).

## Як редагувати контент

Майже весь контент (тарифи, новини, оплата, FAQ, контакти, демо-акаунт)
зосереджено в одному файлі **`src/data/site.js`**.

## Структура

```
src/
  data/site.js        — увесь контент сайту
  components/          — Header, Footer, Logo, Layout, ui (хелпери)
  pages/              — Home, Rates, News, NewsDetail, Contacts,
                        Connection, Payment, Help, Cabinet, NotFound
```
